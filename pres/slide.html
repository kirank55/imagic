How Imagic Works

Project Process & Pipeline
01

Upload & Store

Developers upload raw images to the app. Imagic saves the file details in a **PostgreSQL** database and stores the actual image files in a **Cloudflare R2** cloud bucket.

02

Image Request

A browser requests an image using custom URL settings (like `?width=800&format=webp`). The request goes to a global network of edge servers (**CDN**).

03

Quick Cache Check

The CDN checks if it already has this specific resized image saved. If yes, the image is served instantly to the user (**Cache Hit, under 30ms**).

04

On-Demand Resize

If not saved, the server gets the original from R2, resizes and compresses it instantly on the fly, delivers it to the user, and caches it for future requests.
